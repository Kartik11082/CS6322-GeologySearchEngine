CLUSTER SELECTION DISCUSSION (Association Method)
──────────────────────────────────────────────────────────────────────

The association clustering method selects expansion terms based on co-occurrence
frequency across the local document set (top-50 BM25 results).

Selection Process:
1. For each query stem, compute normalized correlation C(u,v) with all terms
2. Filter candidates using _is_candidate_term():
   - Must be alphabetic, length 3-24 characters
   - Not in stoplist (NLTK + web boilerplate)
   - Document frequency >= 3 and < 5% of corpus
3. Apply IDF weighting: score = C(u,v) × log10(N/df)
4. Select top m_neighbors (4) per query stem
5. Combine and take top max_new_terms (6)

Rationale for selected terms:
- Terms were selected because they have the highest IDF-weighted co-occurrence
  with the query stems across the 50 local documents
- The normalization prevents bias toward high-frequency terms
- IDF weighting ensures discriminative terms are preferred

──────────────────────────────────────────────────────────────────────
CLUSTER SELECTION DISCUSSION (Scalar Method)
──────────────────────────────────────────────────────────────────────

The scalar clustering method selects terms based on cosine similarity of their
association vectors. This captures second-order relationships - terms that
co-occur with similar sets of terms.

Selection Process:
1. Build association matrix: C[u][v] = co-occurrence count
2. Compute cosine similarity: S(u,v) = dot(s_u, s_v) / (|s_u| × |s_v|)
3. Filter candidates (same as association method)
4. Apply IDF weighting
5. Select top neighbors per query stem

Rationale for selected terms:
- Scalar clustering tends to find semantically related terms
- For "metamorphic gneiss formation", terms like "schist", "foliat", "grain"
  are selected because they have similar co-occurrence patterns
- This method is less sensitive to frequency and more to semantic similarity

CLUSTER SELECTION DISCUSSION (Metric Method)
──────────────────────────────────────────────────────────────────────

The metric clustering method selects terms based on physical proximity in the
document text. Terms that frequently appear near query terms are selected.

Selection Process:
1. For each document, find positions of all terms
2. For each query term position, compute 1/distance to all other terms
3. Sum these scores across all documents
4. Filter and apply IDF weighting
5. Select top neighbors (max 4 for metric to reduce noise)
