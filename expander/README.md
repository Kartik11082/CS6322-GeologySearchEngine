# Expander — Geology Search Engine (Student 5)

This module is responsible for **Query Expansion and Relevance Feedback**. It improves poor or short user queries by automatically determining highly correlated terms from a local document set and appending them to the user's search. 

This satisfies the requirements for the Student 5 deliverables.

## Module Reference

### `core.py`

Contains the `QueryExpander` class. It injects itself into Student 2's `SearchEngine` instance to access the inverted index and document store without needing to recreate them in memory.

| Method | Description |
|--------|-------------|
| `expand_rocchio()` | Performs Explicit Relevance Feedback using the Rocchio 1971 formula. Requires manual judgments (`relevant_doc_ids`, `irrelevant_doc_ids`). |
| `expand_association()` | Pseudo-Relevance Feedback (PRF). Builds a local term-document matrix and computes term co-occurrence correlations. |
| `expand_scalar()` | PRF. Computes the cosine similarity of unnormalized association vectors. |
| `expand_metric()` | PRF. Analyzes physical word distance ($1 / r(k_i, k_j)$) within the `text_preview` proxy. |

### `evaluate.py`

An automated experiment script designed to generate the "Before/After" query data required for Section 6 of the final Project Report.

## Implemented Algorithms

This module perfectly maps to the classical IR algorithms covered in the course slides:

1.  **Rocchio Algorithm (Explicit Feedback):** Shifts the initial query vector toward the centroid of explicitly relevant documents and away from non-relevant documents using standard $\alpha, \beta, \gamma$ weighting.
2.  **Association Clusters (Local Analysis):** Calculates term correlation via $c_{u,v} = \sum_{d_j \in D_l} f_{s_u,j} \times f_{s_v,j}$. Expands each query term with its top $m$ co-occurring neighbors within the local document set. 
3.  **Scalar Clusters (Local Analysis):** Calculates the cosine similarity of the association vectors. 
4.  **Metric Clusters (Local Analysis):** Analyzes physical word distance via $c_{u,v} = \sum 1 / r(k_i, k_j)$. Favors terms that appear physically close to query terms within the text.

## How to Run Experiments (For Report)

To generate the data required for your Project Report grading rubric:

```bash
# From the project root, ensure your venv is active
python expander/evaluate.py
```

**What it does:**
1. Loads the 50 queries provided by Student 4 (`cluster_service/benchmarks/queries_50.json`).
2. Runs **Rocchio** on the first 20 queries (simulating manual user judgments).
3. Runs **Association**, **Scalar**, and **Metric** clustering on all 50 queries.
4. Outputs an easy-to-copy file called `expansion_report.txt` containing the Original Query -> Expanded Query pairs.

## Backend Integration

The expansion module is integrated directly into the main FastAPI server (`backend/app.py`).

Start the server:
```bash
python backend/app.py
```

Test it with a POST request:
```bash
curl -X POST [http://127.0.0.1:8000/api/expand](http://127.0.0.1:8000/api/expand) \
     -H "Content-Type: application/json" \
     -d "{\"query\": \"magma\", \"method\": \"association\", \"top_k\": 5}"
```
