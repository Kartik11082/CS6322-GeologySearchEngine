6.1 INTRODUCTION TO QUERY EXPANSION
--------------------------------------------------------------------------------

Query expansion is used to improve search results by reformulating the original 
query with additional terms. This module implements the following local methods:

1. Relevance Feedback (Rocchio Algorithm)
2. Pseudo-Relevance Feedback using three local clustering methods:
   - Association Clustering
   - Metric Clustering  
   - Scalar Clustering

--------------------------------------------------------------------------------
6.2 ROCCHIO RELEVANCE FEEDBACK (20 QUERIES)
--------------------------------------------------------------------------------

The Rocchio algorithm modifies queries based on user relevance judgments using 
the formula:

    q_m = α·q_0 + (β/|D_r|)·Σ(d_j ∈ D_r) - (γ/|D_nr|)·Σ(d_j ∈ D_nr)

Where:
- q_m = modified query vector
- q_0 = original query vector  
- D_r = set of known relevant document vectors
- D_nr = set of known irrelevant document vectors
- α, β, γ = tunable weights

For this project, I used weights α=1.0, β=0.75, and γ=0.25, appending up to 
10 new terms per query.

**Query Selection Criteria:**
I developed 20 queries spanning multiple categories to thoroughly test the 
Rocchio algorithm:
- Single-word geology terms (queries 1-4): volcano, earthquake, fossil, minerals
- Multi-word technical phrases (queries 5-8): magma composition, tectonic plates, 
  sedimentary rock, carbon dating
- Intentional misspellings (queries 9-12): erthsquake, volcnos, fosiils, ignous
- Named geological features (queries 13-16): yellowstone caldera, san andreas 
  fault, hawaiian pahoehoe, mount vesuvius
- Out-of-domain queries (queries 17-20): software engineering, artificial 
  intelligence, stock market trends, football rules

The misspellings test the system's typo-correction capability (using difflib 
similarity matching at 0.82 cutoff). The out-of-domain queries test whether 
expansion can still find tangentially related geology content.

Three local clustering methods were implemented for query expansion. For each 
method, I present 3 example queries showing: (1) the local document set,

==============================
METHOD 1: ASSOCIATION CLUSTERING
==============================

Formula: C(u,v) = Σ(d_j ∈ D_l) [f(u,j) × f(v,j)]
Normalized: C(u,v) / (Σf(u) × Σf(v))

This method computes term co-occurrence correlations across the local document 
set (top-50 BM25 results).

METHOD 2: SCALAR CLUSTERING
==============================

Formula: S(u,v) = (s_u · s_v) / (|s_u| × |s_v|)

Scalar clustering computes cosine similarity between association vectors, 
capturing second-order term relationships.

==============================
METHOD 3: METRIC CLUSTERING
==============================

Formula: C(u,v) = Σ[1 / r(k_i, k_j)]
Where r(k_i, k_j) is the word distance between term positions.

Metric clustering captures proximity-based relationships, finding terms that 
appear physically close to query terms in documents.

**Observations:**
- Association clustering finds statistically co-occurring terms (geographic names)
- Scalar clustering captures semantic similarity (dinosaur↔fossil, aftershock↔earthquake)
- Metric clustering finds contextually adjacent terms (often action-oriented: hunt, sale, buy)

6.10 IMPLEMENTATION DETAILS
--------------------------------------------------------------------------------

The query expansion module is implemented in `expander/core.py` with the 
following key components:

1. **QueryExpander class:** Main interface for all expansion methods
2. **Typo correction:** Using difflib.get_close_matches() with 0.82 cutoff
3. **Stoplist filtering:** Combines NLTK stopwords + web boilerplate terms
4. **IDF weighting:** Expansion terms are scored by TF-IDF discriminativeness
5. **Document frequency filtering:** Terms must appear in 3+ documents but 
   <5% of corpus to avoid noise

**Evaluation Script:** `expander/evaluate.py` generates comprehensive reports 
for all 20 Rocchio queries and 50 PRF queries, saved to `expansion_report.txt`.
